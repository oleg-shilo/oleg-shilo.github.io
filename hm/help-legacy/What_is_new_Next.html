Added //css_include directive

Added another aliases for //css_import directive:
//css_include and //css_inc
They are equivalents of //css_import <file>, preserve_main. These directives are logically similar to the #include in C++.

Added //css_args directive

Now you can specify script command-line arguments directly from code.

Added non-engine directive

The //css_dbg directive allows controlling how the script is loaded in VS2005/VS2005E at execution time. It includes type of the temporary project generated (console/winexe/library) and project debugging arguments. 

Changes in command-line interface

Added new command line argument:
/co:<options>
Pass compiler options directly to the language compiler (eg. /co:/d:TRACE pass /d:TRACE option to C# compiler).

Added new scripts to the Script Library

- css2cs.cs   
   This script converts the cassless C# into standard C#.
- setEV.cs   
   This script allows setting environment variable permanently by using WScript.Shell COM object usually available from VBScript
- createShortcut.cs
   Creates execution shortcut to the script file.
- lib.cs
    Shows Script Library content.

Changes in the CS-Scrip environment configuration

- The format and name of the CS-Script runtime settings file (css_config.dat) is changed. It is now xml file css_config.xml.
- Added HideCompilerWrnings option to control level of details in compiler output information.
- Added option for enabling /disabling the Advanced Shell Extensions from the configuration console.

Updated AsmHelper class

Added InvokeInst method to the AsmHelper class to invoke instance methods. This functionality makes AsmHelper capable of assisting in all possibles dynamic assembly loading/executing scenarios.

Changed CS-Script directory structure

In order to simplify CS-Script deployment architecture Debug folder has been moved from the CS-Script root directory to the root/Lib.
Note: If you are upgrading from any previous version of the CS-Script the content of the previous Debug folder is moved to Lib/Debug/Backup.
